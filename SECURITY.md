# Security policy

NYC Transit Live is a solo-maintained, best-effort public project. There is no
security team, no service-level guarantee, and no bug bounty.

## Reporting a vulnerability

Please report suspected vulnerabilities privately, not in a public issue:

- Preferred: GitHub's private vulnerability reporting for this repository
  (the "Report a vulnerability" button under the Security tab).
- Alternatively: contact the repository owner directly through their GitHub
  profile.

Include enough detail to reproduce the issue. You will get a best-effort
acknowledgement and, where a fix is warranted, a best-effort resolution on a
timeline that fits a hobby project. There is no reward or bounty.

## Scope

This project serves public, read-only transit data and holds no user accounts,
credentials, or personal data. The most useful reports concern the code and its
deployment (for example, a way to make the service serve unsafe content or to
break its isolation), not the upstream transit feeds themselves, which are
operated by their respective agencies.
